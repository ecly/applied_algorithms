Part-of-speech tagger evaluation¶

Preliminaries¶

We assume the taggers are already trained, and the test sets tagged.

Train¶

for lang in cs de en fr hi; do 
    ./tnt/tnt-para -o models/${lang} data/${lang}.train
done

Run¶

for lang in cs de en fr hi; do 
    ./tnt/tnt models/${lang} data/${lang}.test > data/${lang}.test.tagged
done

We typicall evaluate part-of-speech (POS) taggers for:

  1. overall accuracy
  2. accuracy on known vs. unknown words with respect to training data
  3. precision and recall on different POS tags
  4. speed

Datasets¶

Here, we work with five languages: Czech, English, French, German, and Hindi.

Each language bundles three files: LANG.{train|test|test.tagged}.

They respectively contain the training data, manually annotated test data, and tagged test set data.

Reading the data¶

def load_tagged_file(filename):
    """
    Reads the part-of-speech-tagged data in "word[\t]tag[\n]" format
    as a list of sentences, which are in turn lists of <word, tag> pairs.
    """
    unique_words = set()
    sentences = []
    current = []
    with open(filename) as file:
        for line in file:
            line = line. strip()
            if line:
                word, tag = line.split("\t")
                current.append((word, tag))
                unique_words.add(word)
            else:  # empty line is sentence delimiter
                sentences.append(current)
                current = []
    return sentences, unique_words

Calculating the accuracy¶

We define POS tagging accuracy as:

$$\mathrm{accuracy}=\frac{\mathrm{number~of~words~with~correct~POS~tag}}{\mathrm{total~number~of~words}}$$
In [6]:
def get_accuracy(s_data, g_data):
    """
    Caluclates POS tagging accuracy with respect to gold data.
    """
    total = 0
    correct = 0
    for s_sent, g_sent in zip(s_data, g_data):
        for (_, s_tag), (_, g_tag) in zip(s_sent, g_sent):
            total += 1
            if s_tag == g_tag:
                correct += 1
    return correct/total

Known and unknown words¶

First, we get the accuracies separately for words that are:

  1. seen by the taggers at training time
  2. words unique to the test sets
In [24]:
def get_accuracy_known_vs_unknown(s_data, g_data, lexicon):
    """
    Separates the accuracy for known and unknown words.
    Known = exists in the training set.
    """
    correct_known = 0
    correct_unknown = 0
    total_known = 0
    total_unknown = 0
    for s_sent, g_sent in zip(s_data, g_data):
        for (word, s_tag), (_, g_tag) in zip(s_sent, g_sent):
            if word in lexicon:
                total_known += 1
                correct_known += int(s_tag == g_tag)
            else:
                total_unknown += 1
                correct_unknown += int(s_tag == g_tag)
    return correct_known/total_known, correct_unknown/total_unknown

Precision and recall¶

How well does a tagger perform with a specific word type? Say, NOUN, VERB, or ADJ?

Or, in Python code...

In [30]:
def get_precision_and_recall(s_data, g_data, tag):
    """
    Precision and recall for a single POS tag.
    Expressed in terms of true|false positives|negatives.
    """
    true_positives = 0
    true_negatives = 0
    false_positives = 0
    false_negatives = 0
    correct = 0
    for s_sent, g_sent in zip(s_data, g_data):
        for (_, s_tag), (_, g_tag) in zip(s_sent, g_sent):
            if tag == s_tag:  # system predicts tag...
                if s_tag == g_tag:  # ...and the prediction is correct
                    true_positives += 1
                else:  # ...and the prediction is incorrect
                    false_positives += 1
            else:  # system didn't predict tag...
                if tag == g_tag:  # ...and it should have
                    false_negatives += 1
                else:  # ...and it shouldn't have
                    true_negatives += 1
    precision = true_positives/(true_positives + false_positives)
    recall = true_positives/(true_positives + false_negatives)
    return precision, recall
